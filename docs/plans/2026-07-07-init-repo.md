# Plan de Implementación: Inicialización de Repositorio Git y Creación de README

> **Para trabajadores agenticos:** Use `mobiai-mobile-executing-plans-with-subagents` (recomendado) o `mobiai-mobile-executing-plans` para implementar este plan tarea por tarea. Los pasos usan la sintaxis de casillas de verificación para el seguimiento.

**Goal:** Configurar el repositorio Git local para los mods de HumanitZ, documentar detalladamente cada mod en un archivo README.md en español y subir todo el código al repositorio remoto de GitHub en https://github.com/SamiGamin/HUMANHOSTMODS.

**Architecture:** Se creará un archivo `.gitignore` optimizado para proyectos C#/.NET y BepInEx para no subir archivos basura de compilación (`bin`, `obj`, archivos de log, etc.). Se creará un `README.md` exhaustivo y estético describiendo el propósito, configuración y controles de cada uno de los 5 mods. Luego se inicializará Git localmente, se añadirá el remote y se subirá la rama principal a GitHub.

**Tech Stack:** Git, Markdown, C# (.NET/BepInEx/Harmony)

**Platform:** PC / Unity Modding (HumanitZ)

### Tarea 2: Creación del archivo README.md

**Archivos:**
- Crear: `README.md`

- [ ] **Paso 1: Redactar la documentación en español para cada mod**
  Escribir un documento explicativo que contenga:
  - Título principal del repositorio.
  - Descripción general del paquete de mods para el juego **HumanitZ**.
  - Índice de los mods incluidos.
  - Sección detallada para cada uno de los 5 mods (`DeveloperItemSpawner`, `GatheringMultiplier`, `MiModDeAlmacenamiento`, `StackCustomizer`, `radar`):
    - **DeveloperItemSpawner** (Consola de Spawneo de Ítems): Explicar su menú GUI con `F10`, el extractor de catálogo con `F9`, la traducción en caliente y la selección de cantidad de ítems.
    - **GatheringMultiplier** (Multiplicador de Recolección): Explicar la multiplicación de madera al talar y de minerales/plantas al minar, configurable en `humanhost.gathering.multiplier.cfg` (por defecto 3.0x).
    - **MiModDeAlmacenamiento** (Expansión de Cofres): Explicar la modificación independiente de filas y columnas de cofres Tier 1 (Madera), Tier 2 (Metal) y maletas, configurable en `humanhost.cofres.cfg`.
    - **StackCustomizer** (Tamaño de Stacks): Explicar el cambio del tamaño máximo de stack (por defecto 999), stack ilimitado y forzar que todos los ítems sean apilables, configurable en `humanhost.stack.tamano.cfg`.
    - **radar** (Human Host Radar / RadarSniffer): Explicar la herramienta de depuración con Raycast en `F` para volcar componentes de Unity por reflexión y la tecla `Tab` para comprobar menús activos.
  - Instrucciones de instalación general usando BepInEx.

- [ ] **Paso 2: Revisar la ortografía y formato markdown**
  Comprobar que el archivo se visualice correctamente con un diseño limpio y profesional.
